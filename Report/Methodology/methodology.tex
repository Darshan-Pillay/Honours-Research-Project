\section{Dataset}
The PlantVillage dataset is used in our study. PlantVillage is a collection of 54309 annotated diseased and healthy leaf images spanning several plant species, including tomato \cite{hughesOpenAccessRepository2016}. The size of the subset of tomato leaf images is 18162. Several images of each tomato leaf were captured outside under natural light in various lighting conditions and orientations, against a black or grey background, using a standard digital camera \cite{hughesOpenAccessRepository2016, kiranPlantLeafDisease2023}. The images are captured in the RGB colour space and have dimensions 256x256. We have selected PlantVillage as our primary dataset because it is widely used in image processing research for plant disease detection \cite{liReviewComputerVision2020, jiangConvolutionalNeuralNetworks2020, jafarRevolutionizingAgricultureArtificial2024, g.IdentificationPlantLeaf2019, francisDiseaseDetectionClassification2019, govardhanDiagnosisTomatoPlant2019}. Using this dataset will allow future researchers to compare our proposed method to previous studies.

\begin{itemize}
	\item Texture features have proven to be espeically useful for plant pathogen detection \cite{arivazhaganDetectionUnhealthyRegion2013, kimClassificationGrapefruitPeel}
	\item Textture features are emperically best predictors of tomotoe leaf disease detection \cite{mokhtarSVMBasedDetectionTomato2015} since occurenc of plat disaes cause specific changes in texture of the leaft and therefeore textetd based features can be used to train a classifier \cite{ahmedLeafImagebasedPlant2021}
	\item The gray level co-occurence methodlogyy is a statisitcal way to descibe the shape by smapling the way gray levels ocuren in relation to other gray levels. And have been applied successfully to detect and clasify plant pathogens \cite{arivazhaganDetectionUnhealthyRegion2013, mokhtarSVMBasedDetectionTomato2015}.
	\item Caculate gray level co-occurence matrix. Concerned with the spatial statistical distrubiton of gray tones. Based on assumption that the texture information in the overall average spatial relationship the gray tones in the image have with one another. \cite{haralickTexturalFeaturesImage1973}. Following a similar approach to \cite{mokhtarSVMBasedDetectionTomato2015} we compute the common energy, contrast, sum of squares, correlation, entropy, sum entropy, cluster shade, and cluster prominence, and homogenity metrics from the gray level occurence matrix. These are defined as well in the original paper introducing GLCM and mockhtar.
\end{itemize}

\section{Classification  Mechanism}
\begin{itemize}
	\item SVM is a machine supervised learning method which has been shown to be an effective classifer of leaf diseases in an image processing pipeline \cite{mokhtarSVMBasedDetectionTomato2015}.
	\item We use machine learning Support Vector Machine binary classifier to identify whether or not a leaf is diseased or not. This has shown to be particularly effective for leaf disease detection in the literature \cite{kiranPlantLeafDisease2023, mokhtarSVMBasedDetectionTomato2015, harakannanavarPlantLeafDisease2022, govardhanDiagnosisTomatoPlant2019}.
	\item We pass feature vector calculated from gray level co-occurence matrix
	\item This feature vector is fed into SVM classifier which determines whether or not original image leaf has disease or not
\end{itemize}

\section{Microservice architecture}
\begin{itemize}
	\item Use of dockerized containers allows reproducibility of our development environemnt on varying hardwares. Thus allowing computatinal repdoruciblity and other researchers to easily extend and reproduce our results with less effort \cite{boettigerIntroductionDockerReproducible2015}.
	\item A microservice architecture is a pattern of structuing a computing system application as a composition of microservices. A microservice is an independnet unit of the application which can be deployed run and scaled in isolation. Some benefits include modulatiry in that parts of the system can be replaced or extended without resambelling the whole system or having all the knowledge of the system. \cite{dragoniMicroservicesYesterdayToday2017, velepuchaSurveyMicroservicesArchitecture2023}
	\item Each service can be developed independelty and therefore choose the most suitable tool chains to solve the problem. There exsit numerous open-source software packages that can be used in plant phenotyping and the ability to implement a solution in any one of them and repalce them in the future is of great benefit to researchers. \cite{dragoniMicroservicesYesterdayToday2017} (benefits of microservice approach in this cite)
	\item Our image processing pipeline consists of a cluster of docker nodes running image processing services
	\item We use a dockerized ubuntu container with python installation as nodes
	\item Each node runs a flask rest API web service
	\item Reason for this choice is that we can implement on-site local network of nodes that dont rely on a cloud or central server (edge networks). Therefore keeping cost low \cite{grecoComputerVisionAlgorithms2024}
\end{itemize}
